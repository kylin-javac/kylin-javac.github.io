linux 下的简单命令和git版本管理控制工具的使用

Posted on 2018-04-05
Words count in article: 1,044 | Reading time ≈ 4

内容可能不是很深入详细，但是我每天都要用到的命令,抽了个时间简单的整理了下

1.linux 下的简单命令

  1. cd .. #返回上一级目录

  2. cd ~ #返回当前用户的根目录

  3. cd // #返回系统的跟目录

  4. . ./xxx.sh #执行shell脚本

  5. chmod 777 file #将系统的最高权限授予这个文件,root 用户组 当前用户 读 写 可执行 每个用户都有后面三种权限

  6. chmod 777 -r filename #将系统的最高权限授予这个文件以及该文件夹下的子文件(递归到单个文件)

  7. cp 目标.file 复制到哪里的路径 #复制单个文件

  8. cp -r 目标.filename 复制到哪里的路径 #复制整个目录下的文件

  9. scp -r 用户名@主机地址:文件或文件夹 目标地址 #复制远程计算机中的文件到本地计算机中

  10. tar -zxvf xxx.gz -C /解压路径 #将压缩包解压到指定路径

  11. tar -cxvf 文件名.tar 需要打包的文件或文件夹路径 #将文件或文件夹打包

  12. ssh 用户名@主机地址 #在ubuntu终端中登录远程主机

  13. vim 的操作
    1. esc #这是一个万能键 他可以在任何模式下进行切换
    2.i #由命令模式进入编辑模式并将光标定位当前位置
    a #由命令模式进入编辑模式并将光标定位当前位置的后一位
    o #由命令模式进入编辑模式另一起行将光标定位到行首
    3. 命令模式中
    h #光标向右移一个位置
    l #光标向左移一个位置
    k #光标向上移动一行
    j #光标向下移动一行
    u #撤回
    yy #复制当前行
    p #粘贴剪贴版中的内容
    /str #查找相应的字符 找到后按n跳转到下一个匹配处
    /vsp file #可以竖屏显示的方式打开其他文本文件,这招很好用
    尤其是比对两个文件时

  14. 有一些常用的服务需要配置比如sshServer端，ftpServer端
    sambaServer端，当然还有如何安装linux.这些过程都需要截图才能
    表达清楚，不然只粘贴命令会让人不知所云。
    用linux开发一般都会装sshServer和sambaServer，sambaServer用来适应windows环
    境(毕竟有些人还是喜欢用鼠标)sshSever 则可以用来在服务器上修改代码和编译软件

2.git版本管理控制命令的使用

git 的命令很简单 但结合着Androi源码这样的工程去用会让你对他独自发叹(道行不深，资历太浅)

  1. git status # 当前仓库的状态，哪些文件已经更改,哪些文件未添加进来

  2. git diff # 当前仓库状态中的文件和上一次提交代码时文件做了哪些改动,改动的异同点都会标出来

  3. git add file # 将更改或未添加的文件添加到仓库中区

  4. git rm file # 将仓库中原有的文件移除出仓库

  5. git commit -m “更改内容说明” # 将文件提交到本地仓库

  6. git pull 远程仓库地址 #拉取远程服务器中的代码，合并冲突
    (多说一句 这种方式只有其实是让git帮我们合并代码,
    我们自己不merge，只有双方都改了同一个地方，git才会报出冲突警告，
    其他的他就自己干了，这样其实有优点也有缺点,这个坑比较深,,,,,,)

  7. git clone ssh://用户名@主机地址/仓库地址 #克隆远程服务器仓库代码

  8. git pull ssh://用户名@主机地址/仓库地址 #拉取远程服务器仓库代码

3.adb 命令的简单使用

  1. adb remount #重新挂载一下Android主机
  2. adb push loca.apk data/appi #将apk推送到Android data/app目录
  3. adb shell reboot #重启Android机器
  4. adb shell #进入Android主机的shell环境
  5. adb shell am startService 应用名/包名.类名 #手动启动服务
